## 2. 系统当前定位

### 2.1 当前系统解决的问题

本系统当前用于解决同城配送加盟体系的日常经营分析问题，重点支持：

- 全国加盟商经营表现对比
- 单城市合伙人经营分析
- 直营专项经营分析
- 新骑手、新商家、新合伙人表现跟踪
- 小时运力与高峰承接分析
- 补贴、收入、提成、经营利润分析
- 风险加盟商、关注加盟商识别
- 日报、周报、月报管理看板的指标底座

### 2.2 当前系统定位

当前系统是一个：

> **本地单机版经营分析系统**

它不是一个完整的数据平台，也不是财务系统。  
当前重点是“先准确、先可用、先能支持业务动作”，而不是先做复杂平台化能力。

### 2.3 当前已落地范围

当前已落地：

- 四类数据自动导入：订单、骑手、商家、合伙人
- ODS / DWD / ADS 三层建模
- DuckDB 本地分析底座
- 全国页、单城市页、直营页三页结构
- 核心口径计算：有效订单、新主体、补贴、利润、活跃主体、SLA
- 风险清单、波动预警、加盟商分层、小时运力分析

当前未落地或仅留扩展位：

- 真正在线骑手数
- 客诉率
- 正式异常率事件表
- 拜访商家台账
- 总部动作执行率
- 固定成本与财务净利润
- 盈亏平衡单量
- 多人协作权限系统

- `province`
- `city`
- `district`

当前支持格式包括：

- `广东省-广州市-天河区`
- `广东省/广州市/天河区`
- `广东省 广州市 天河区`
- `广东省广州市天河区`
- `北京市朝阳区`
- `广西壮族自治区南宁市青秀区`

当前 DWD 已覆盖的关键订单字段包括：

- 用户ID
- 骑手ID、骑手名称
- 在职状态、在职类型
- 下单来源

- 订单状态
- 下单、支付、接单、取消、完成时间
- 下单日期、下单小时、接单小时
- 实付金额、合伙人收入、骑手提成

## 5. 正式业务口径

### 5.1 主统计日期

- 主看板统计日期统一按：`下单日期`

### 5.2 有效订单

```text
有效订单 = 完成订单 + 支付后取消且支付到取消时长 > 阈值分钟
```

默认阈值：

- `5 分钟`

说明：

- 正好 `5 分钟` 不计入有效订单
- 未支付取消不计入有效订单

### 5.3 有效取消订单

```text
有效取消订单 = 支付后取消且支付到取消时长 > 阈值分钟
```

### 5.4 超时取消

```text
超时取消 = 下单到取消时长 > 阈值分钟
```

### 5.5 完成率与取消率

主看板统一口径：

- `完成率 = 完成单量 / 总单量`
- `取消率 = 取消单量 / 总单量`

### 5.6 有效订单完成率与有效订单取消率

- `有效订单完成率 = 完成单量 / 有效订单量`
- `有效订单取消率 = 有效取消单量 / 有效订单量`

### 5.7 新主体

- 新骑手：按入职时间，默认窗口 `30 天`
- 新商家：按注册时间，默认窗口 `30 天`
- 新合伙人：按开城时间，默认窗口 `90 天`

新合伙人表现当前要求单独区分：

- `30 日表现`
- `60 日表现`
- `90 日表现`

### 5.8 活跃主体

当前活跃主体统一定义为：

- 当前筛选范围内，`完成单量 >= 动态阈值`

适用于：

- 活跃合伙人
- 活跃骑手
- 活跃商家

### 5.9 高峰接单骑手数

当前统一用：

- `高峰接单骑手数`

替代：

- `高峰在线骑手数`

说明：

- 当前系统能算的是“该时段实际接单的去重骑手数”
- 这不是严格意义上的在线骑手数

### 5.10 准时率

```text
准时率 = 接单后 30 分钟内完成订单 / 完成订单
```

说明：

- 仅统计完成订单
- 缺少接单时间或完成时间的订单不计入分母

### 5.11 SLA 履约率与超时率

当前系统同时保留一套可配置 SLA 指标：

- `SLA履约率 = 接单后在配置时长内完成订单 / 完成订单`
- `SLA超时率 = 接单后超过配置时长完成订单 / 完成订单`

规则：

- 按合伙人可配置时长
- 默认回退 `30 分钟`

### 5.12 收益相关口径

- 实际收款总额：`完成订单实付金额合计`
- 合伙人收入总额：`完成订单合伙人收入合计`
- 合伙人补贴总额：`完成订单合伙人补贴合计`
- 骑手提成总计：`完成订单骑手提成合计`
- 经营利润：`合伙人收入总额 - 合伙人补贴总额`
- 客单价：`完成订单实付金额合计 / 完成单量`
- 骑手单均提成：`骑手提成总计 / 完成单量`
- 人效：`完成单量 / 骑手人数`

### 5.13 命名统一

- `骑手佣金 = 骑手提成`
- `骑手单均收入` 不再单独保留
- 统一显示为：`骑手单均提成`
- `月净利润 / 亏损` 当前统一改名为：`经营利润`

## 15. 附录

### 15.1 当前正式口径简表

| 指标 | 当前正式口径 |
| --- | --- |
| 主统计日期 | 下单日期 |
| 有效订单 | 完成订单 + 支付后取消且支付到取消时长 > 阈值 |
| 有效取消订单 | 支付后取消且支付到取消时长 > 阈值 |
| 超时取消 | 下单到取消时长 > 阈值 |
| 完成率 | 完成单 / 总单 |
| 取消率 | 取消单 / 总单 |
| 有效订单完成率 | 完成单 / 有效订单 |
| 有效订单取消率 | 有效取消单 / 有效订单 |
| 准时率 | 接单后 30 分钟内完成订单 / 完成订单 |
| SLA履约率 | 接单后在配置时长内完成订单 / 完成订单 |
| 经营利润 | 合伙人收入总额 - 合伙人补贴总额 |
| 骑手提成总计 | 完成订单骑手提成合计 |
| 骑手单均提成 | 骑手提成总计 / 完成单量 |
| 人效 | 完成单量 / 骑手人数 |
