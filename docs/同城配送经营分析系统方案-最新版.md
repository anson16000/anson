# 同城配送经营分析系统方案（最新版）

**文档定位**：当前系统唯一主方案文档
**适用对象**：业务负责人、运营负责人、数据分析、后续开发与维护人员
**当前版本日期**：2026-04-19
**项目路径**：`F:\codex\delivery-dashboard`

---

## 1. 系统定位

当前系统用于支撑同城配送加盟体系的经营分析，重点覆盖：

- 全国经营总览
- 单城市经营分析
- 时段热力与履约分析
- 主体结构与明细分析
- 风险与预警分析

当前系统是一个**本地单机经营分析系统**，不是完整的数据平台或财务系统。

---

## 2. 技术架构

- 后端：Python + FastAPI + SQLAlchemy
- 数据底座：DuckDB
- 前端：原生 HTML / CSS / JS
- 数据模型：ODS / DWD / ADS
- 当前统一端口：`8090`

页面入口：

- `/` 全国总览
- `/partner` 城市经营
- `/partner/hourly` 时段热力与履约
- `/partner/entities` 主体分析
- `/alerts` 诊断预警
- `/direct` 兼容跳转到 `/partner?section=direct`

---

## 3. 正式业务口径

- 主统计日期：按下单日期
- 有效订单：完成订单 + 支付后取消且支付到取消时长大于阈值分钟
- 超时取消：下单到取消时长大于阈值分钟
- 完成率：完成订单 / 总订单
- 取消率：取消订单 / 总订单
- 有效订单完成率：完成订单 / 有效订单
- 有效订单取消率：有效取消订单 / 有效订单
- 活跃主体：按完成单量阈值动态定义
- 新合伙人表现：区分 30 / 60 / 90 日
- 加盟商分层：按日均单量
- 实际收款总额：完成订单实付金额合计
- 合伙人利润：合伙人收入总额 - 合伙人补贴总额
- 骑手提成总计：完成订单骑手提成合计
- 骑手单均提成：骑手提成总计 / 完成订单
- 准时率：接单后 30 分钟内完成订单 / 完成订单
- SLA 履约率 / 超时率：按合伙人配置时长
- 高峰接单骑手数：替代“高峰在线骑手数”
