# 同城配送经营分析系统字段字典 v1

## 1. 文档目的
本字典用于统一同城配送经营分析系统的字段命名、来源关系、指标定义和计算规则，作为以下工作的唯一参考底稿：

- 数据导入与字段映射
- 经营分析口径对齐
- 页面指标解释
- 问题排查与后续扩展

## 2. 正式口径
以下三条为当前系统正式默认口径：

1. 主看板统计日期：按`下单日期`统计。
2. 有效订单：`完成订单 + 支付后取消且支付到取消时长 > 5 分钟的订单`。
3. 超时取消：`下单到取消时长 > 5 分钟的取消订单`。

相关衍生口径同步固定为：

- 有效取消订单：`支付后取消且支付到取消时长 > 5 分钟`
- 有效订单取消率：`有效取消订单 / 有效订单`
- 客服在线：`下单时间在 09:00-20:00`
- 客服在线超时无人接取消：`客服在线 + 无人接 + 下单到取消时长 > 5 分钟`

## 3. 数据分层

| 层级 | 说明 |
| --- | --- |
| ODS | 原始接入层，保留原始订单和名单字段 |
| 标准名单层 | 骑手、商家、合伙人标准主数据 |
| DWD | 标准订单明细层，承接所有经营口径 |
| ADS | 面向页面展示和经营分析的汇总层 |

## 4. 主关联关系

| 主体 | 关联规则 |
| --- | --- |
| 订单 -> 骑手 | `订单明细.配送员ID = 帮手信息.帮手ID` |
| 订单 -> 商家 | `订单明细.商家ID = 商户信息.商家ID` |
| 订单 -> 合伙人 | `订单明细.合伙人ID = 合伙人信息.ID` |
| 区域维度 | 优先取合伙人名单解析后的 `province / city / district` |

### 6.5 核心经营口径字段

| 标准字段名 | 中文名称 | 字段类型 | 来源 | 业务含义 | 计算规则 |
| --- | --- | --- | --- | --- | --- |
| `pay_cancel_minutes` | 支付到取消时长(分钟) | float | 衍生 | 是否有效取消判断依据 | `(cancel_time - pay_time) / 60` |
| `order_elapsed_minutes_to_cancel` | 下单到取消时长(分钟) | float | 衍生 | 是否超时取消判断依据 | `(cancel_time - create_time) / 60` |
| `is_valid_order` | 是否有效订单 | bool | 衍生 | 进入履约统计的订单 | `完成订单` 或 `支付后取消且 pay_cancel_minutes > 5` |
| `is_valid_cancel_order` | 是否有效取消订单 | bool | 衍生 | 有效订单中的取消部分 | `is_cancelled AND pay_cancel_minutes > 5` |
| `is_timeout_cancel` | 是否超时取消 | bool | 衍生 | 用户等待过久后取消 | `is_cancelled AND order_elapsed_minutes_to_cancel > 5` |
| `is_not_timeout_cancel` | 是否未超时取消 | bool | 衍生 | 5 分钟内取消 | `is_cancelled AND order_elapsed_minutes_to_cancel <= 5` 或无法算出时长 |
| `is_unaccepted_cancel` | 是否无人接取消 | bool | 衍生 | 接单前取消 | `is_cancelled AND accept_time IS NULL` |
| `is_accepted_cancel` | 是否接单后取消 | bool | 衍生 | 接单后取消 | `is_cancelled AND accept_time IS NOT NULL` |
| `is_rider_noliability_cancel` | 是否无责取消 | bool | 衍生 | 接单后无责取消 | `is_cancelled AND order_status LIKE '%无责%'` |
| `is_new_rider_order` | 是否新骑手订单 | bool | 衍生 | 新骑手观察窗口内订单 | 下单日期距入职日期在 `0-30` 天 |
| `is_new_merchant_order` | 是否新商家订单 | bool | 衍生 | 新商家观察窗口内订单 | 下单日期距注册日期在 `0-30` 天 |
| `is_new_partner_order` | 是否新合伙人订单 | bool | 衍生 | 新合伙人观察窗口内订单 | 下单日期距开城日期在 `0-90` 天 |
| `hq_subsidy_amount` | 总部补贴金额 | float | 衍生 | 总部承担的优惠 | 有总部优惠金额且无营销优惠券ID |
| `partner_subsidy_amount` | 合伙人补贴金额 | float | 衍生 | 合伙人承担的优惠 | `优惠金额`；若有营销券则总部优惠也计入合伙人补贴 |
| `is_cross_day_order` | 是否跨天单 | bool | 衍生 | 下单与完成是否跨天 | `DATE(create_time) <> DATE(complete_time)` |

## 7. ADS 指标口径

### 7.1 全国页核心指标

| 指标 | 口径 |
| --- | --- |
| 总订单 | 所选范围内订单总数 |
| 有效订单 | `is_valid_order = true` 的订单数 |
| 完成订单 | `is_completed = true` 的订单数 |
| 取消订单 | `is_cancelled = true` 的订单数 |
| 完成率 | `完成订单 / 有效订单` |
| 活跃合伙人数 | 有订单的去重合伙人数 |
| 新合伙人数 | 有新合伙人订单的去重合伙人数 |
| 活跃商家数 | 有订单的去重商家数 |
| 新商家数 | 有新商家订单的去重商家数 |
| 活跃骑手数 | 有订单的去重骑手数 |
| 新骑手数 | 有新骑手订单的去重骑手数 |
| 总部补贴 | `hq_subsidy_amount` 汇总 |
| 合伙人补贴 | `partner_subsidy_amount` 汇总 |

### 7.2 单合伙人城市页核心指标

| 指标 | 口径 |
| --- | --- |
| 城市总订单 | 单合伙人范围内订单总数 |
| 城市有效订单 | `is_valid_order = true` |
| 城市完成订单 | `is_completed = true` |
| 城市取消订单 | `is_cancelled = true` |
| 城市完成率 | `完成订单 / 有效订单` |
| 城市取消率 | `取消订单 / 总订单` |
| 新骑手完成单量 | `is_new_rider_order = true AND is_completed = true` |
| 新商家完成单量 | `is_new_merchant_order = true AND is_completed = true` |

### 7.3 直营经营分析指标

| 指标 | 口径 |
| --- | --- |
| 有效订单 | `is_valid_order = true` |
| 有效取消订单 | `is_valid_cancel_order = true` |
| 有效订单取消率 | `有效取消订单 / 有效订单` |
| 无人接取消 | `is_unaccepted_cancel = true` |
| 接单后取消 | `is_accepted_cancel = true` |
| 无责取消 | `is_rider_noliability_cancel = true` |
| 超时取消 | `is_timeout_cancel = true` |
| 客服在线超时无人接取消 | `service_online_flag = true AND is_timeout_cancel = true AND is_unaccepted_cancel = true` |
| 券单量 | `has_coupon_order = true` |
| 券优惠金额 | `hq_discount_value + discount_value` |
| 兼职人效 | `兼职完成单量 / 兼职接单骑手数` |
| 全职人效 | `全职完成单量 / 全职接单骑手数` |

## 8. 商家型用户口径
商家型用户不在 DWD 固化，当前在查询层动态计算：

- 默认阈值：`20 单`
- 判定规则：`某用户完成单量 >= 阈值`

## 9. 注意事项

1. 本字典以当前代码实现为准，而不是历史方案文档的早期版本。
2. 若后续调整 5 分钟阈值、新主体观察窗口或补贴归属规则，必须同步更新本字典。
3. 旧历史导出文件可能存在乱码表头，运行时已通过字段别名映射兼容，但后续应优先推动源数据表头标准化。
