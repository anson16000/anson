# Python + DuckDB + Parquet + Power BI 重构方案

版本：v1.1

日期：2026-05-12

## 1. 目标

Power BI 不再直连 DuckDB，而是读取 Parquet 成品文件。

最终链路：

```text
原始订单 / 骑手 / 商家 / 合伙人文件
-> Python 导入
-> DuckDB ODS / DWD / ADS
-> 导出中文 Parquet 成品文件
-> Power BI 读取 Parquet
```

本版已把 Power BI 看到的文件名和字段名改成中文，降低使用门槛。

## 2. 导出命令

导入数据成功后会自动导出 Parquet。

也可以手动导出：

```powershell
python main.py export-powerbi
```

或双击：

```text
04-导出PowerBI-Parquet.bat
```

默认导出目录：

```text
exports/powerbi_parquet/
```

## 3. Power BI 读取哪些文件

### 维度表

| 文件名 | 作用 |
|---|---|
| `日期维度.parquet` | 日期、年、月、日、星期 |
| `合伙人维度.parquet` | 合伙人名称、省、市、区县、开通日期 |
| `骑手维度.parquet` | 骑手姓名、入职日期、状态 |
| `商家维度.parquet` | 商户名称、店铺名称、注册日期、状态 |

### 事实表

| 文件名 | 作用 |
|---|---|
| `全国日汇总.parquet` | 全国和区域每天经营情况 |
| `合伙人日汇总.parquet` | 合伙人每天经营情况，Power BI 主表 |
| `小时运力汇总.parquet` | 小时单量、接单骑手数、全职/兼职人效 |
| `骑手日汇总.parquet` | 骑手每天完成单量 |
| `商家日汇总.parquet` | 商家每天完成单量 |
| `订单明细事实表.parquet` | 订单级联动事实表，用来串联日期、合伙人、商家、骑手和订单来源 |
| `订单来源汇总.parquet` | 各订单来源每天单量 |
| `用户商家识别汇总.parquet` | 用户和商家主体识别基础数据 |

### 诊断表

| 文件名 | 作用 |
|---|---|
| `合伙人健康度.parquet` | 合伙人健康状态、完成率、取消率、人效 |
| `合伙人波动预警.parquet` | 每日完成单量波动 |
| `合伙人风险建议.parquet` | 风险对象、优先级和建议动作 |
| `数据版本.parquet` | 当前数据版本和发布时间 |

## 4. 主要中文字段

## 7. 常用 DAX

完成单量：

```DAX
完成单量 =
SUM('合伙人日汇总'[完成订单])
```

总订单：

```DAX
总订单 =
SUM('合伙人日汇总'[总订单])
```

完成率：

```DAX
完成率 =
DIVIDE([完成单量], [总订单])
```

活跃骑手数：

```DAX
活跃骑手数 =
SUM('合伙人日汇总'[活跃骑手数])
```

人效：

```DAX
人效 =
DIVIDE([完成单量], [活跃骑手数])
```

经营利润：

```DAX
经营利润 =
SUM('合伙人日汇总'[经营利润])
```

本月截止完成单量：

```DAX
本月截止完成单量 =
VAR MaxFactDate = MAX('合伙人日汇总'[日期])
VAR MonthStart = DATE(YEAR(MaxFactDate), MONTH(MaxFactDate), 1)
RETURN
CALCULATE(
    [完成单量],
    DATESBETWEEN('日期维度'[日期], MonthStart, MaxFactDate)
)
```

上月同期完成单量：

```DAX
上月同期完成单量 =
VAR MaxFactDate = MAX('合伙人日汇总'[日期])
VAR ThisMonthStart = DATE(YEAR(MaxFactDate), MONTH(MaxFactDate), 1)
VAR LastMonthStart = EDATE(ThisMonthStart, -1)
VAR LastMonthEnd = EDATE(MaxFactDate, -1)
RETURN
CALCULATE(
    [完成单量],
    REMOVEFILTERS('日期维度'),
    DATESBETWEEN('日期维度'[日期], LastMonthStart, LastMonthEnd)
)
```

同期差值：

```DAX
同期差值 =
[本月截止完成单量] - [上月同期完成单量]
```

同期增长率：

```DAX
同期增长率 =
DIVIDE([同期差值], [上月同期完成单量])
```

## 8. 刷新流程

日常刷新：

```text
1. 放入新的原始数据文件
2. 运行 01-一键导入数据.bat
3. 程序自动重建 DuckDB 并导出 Parquet
4. 打开 Power BI
5. 点击刷新
```

只重新导出 Parquet：

```text
1. 运行 04-导出PowerBI-Parquet.bat
2. 打开 Power BI
3. 点击刷新
```

## 9. 验收清单

- `exports/powerbi_parquet/` 下能看到中文文件名的 Parquet 文件。
- Power BI 字段面板里显示中文字段名。
- `合伙人日汇总.parquet` 的完成订单与网页看板一致。
- `小时运力汇总.parquet` 有全职/兼职接单骑手数和人效。
- `订单来源汇总.parquet` 能展示订单来源。
- Power BI 不再使用 DuckDB ODBC。
- Power BI 刷新时不再报 DuckDB 文件被占用。
