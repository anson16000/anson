# 3 页重构版改版差异说明

## 这次为什么要改
当前系统虽然已经具备全国、单城市、直营三张页面，但仍存在 4 个问题：

1. 三页职责重叠，尤其是单城市页和直营页
2. `merchant_like_threshold` 作为全局参数污染其他模块
3. 单次查询时长缺少统一上限
4. 局部筛选经常触发整页刷新，影响使用体验和后续拆页

第一阶段改版的目标，不是直接做 5 页，而是先把现有 3 页改成未来可拆的“模块化母版”。

## 这次真正改了什么
### 1. 主指标统一
三页首屏统一只保留以下主指标：
- 总订单
- 有效订单
- 完成订单
- 取消订单
- 完成率
- 活跃骑手数
- 新骑手数
- 活跃商家数
- 新商家数
- 总部补贴
- 合伙人补贴

### 2. 查询上限统一
- 所有页面单次查询最多 31 天
- 前端先拦截
- 后端再兜底拦截
- 超限统一提示

### 3. 商家型用户阈值收口
- `merchant_like_threshold` 不再出现在单城市页全局筛选
- 只保留在“用户主体识别”模块
- 只刷新该模块，不影响整页其他统计

### 4. 局部筛选规则明确
- 名单筛选只影响名单明细
- 主体识别阈值只影响主体识别
- 不允许局部筛选污染全页统计

### 5. 页面去重
- 单城市页继续承载通用城市经营分析
- 直营页只保留直营专项内容
- 骑手 / 商家小表不再既在图下重复，又在名单区重复

## 三页改版后的职责
### 全国页
保留：
- 核心摘要
- 全国趋势
- 区域排名
- 加盟商日均单量分层
- 关注加盟商
- 风险加盟商
- 新合伙人表现
- 波动预警

弱化：
- 小时运力概览只保留摘要，不做主场分析

### 单城市页
保留：
- 经营摘要
- 经营收益
- 小时运力
- 热力图
- 新骑手日单量
- 新商家日单量
- 用户主体识别
- 名单明细

变化：
- 新骑手 / 新商家改成贡献摘要 + 趋势
- 名单明细独立成区
- 名单筛选独立成局部筛选

### 直营页
保留：
- 总览卡片
- 日单量统计
- 有效订单统计
- 时间段数据
- 渠道对比
- 优惠金额统计
- 收益指标

删减：
- 热力图
- 通用名单
- 通用主体识别
- 通用城市经营摘要

## 对业务侧的直接影响
- 页面会更清楚，三页职责不再打架
- 单城市页将成为未来“时段热力与履约”“主体分析”两页的母版
- 直营页更聚焦，不再像第二个单城市页
- 后续从 3 页拆成 5 页时，返工量会明显降低

## 第一阶段验收重点
1. 三页首屏主指标是否完全统一
2. 单城市页与直营页是否明显去重
3. 主体识别阈值是否只影响主体识别模块
4. 名单筛选是否只影响名单明细
5. 所有页面是否都限制最多 31 天
6. 直营页是否只剩直营专项内容
